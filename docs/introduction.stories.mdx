Biblioteket er optimeret til at gøre det så nemt for bundlers som muligt, at anvende tree shaking.
Ligeledes er biblioteket optimeret til at gøre det nemt for en udvikler at udvikle og udgive nye

## Install

Komponentbiblioteket er skrevet i React. Biblioteket består af flere pakker, hvor "grundpakkerne"
er:

<ul>
  <li>Hooks</li>
  <li>Base components</li>
  <li>Utils</li>
</ul>

Disse pakker kan installeres og bruges uafhængigt af hinanden. Hvis man i et projekt eksempelvis kun
har brug for utils, så kan man nøjes med at installere pakken "utils". Det samme princip gælder for
resten af pakkerne.

#### base-components

Base-components består af basale funktionelle komponenter, der ikke er stylet (med CSS eller lign.).
Formålet med denne pakke er at udstille komponenter og funktionalitet, der "normalvis" ikke
eksisterer i HTML, men som er brugbart på tværs af forskellige applikationer (uafhængig af styling).
Et eksempel er en Accordion. Der er ikke et HTML tag, der hedder en "Accordion", som blot har
funktionaliteten som en "Accordion" (og som ikke er stylet). Modsat en "Button". Her findes der i
HTML et "Button" tag, hvor en Button har forskellige events og attributter. Base-components
udstiller da ikke-stylede komponenter, med "udvidet" funktionalitet. Stylingen af de forskellige
komponenter er op til klienten, der bruger denne pakke.

<Source
  language="bash"
  code={dedent`
  npm install --save @norlys/base-components
  `}
/>

#### utils

Utils udstiller hjælpefunktioner, der er nyttige på tværs af projekter.

<Source
  language="bash"
  code={dedent`
  npm install --save @norlys/utils
  `}
/>

#### hooks

Hooks pakken udstiller React Hooks, der er nyttige på tværs af projekter. En hook, der er brugbar på
tværs af applikationer er eksempelvis: "use-local-storage" eller "use-form". Formålet med pakken er
at udstille "hjælpe-hooks".

<Source
  language="bash"
  code={dedent`
  npm install --save @norlys/hooks
  `}
/>

### Specialiserede pakker (styling-specifik)

Til sidst er der de specialicerede pakker, som har "base-components" som afhængighed. Formålet med
denne pakke er at udstille brand-specifikke komponenter. Dvs. stylede komponenter. Derved skal de
funktionelle komponenter fra "base-components" styles.

<Source
  language="bash"
  code={dedent`
  npm install --save @norlys/norlys-components
  `}
/>

### Use

Komponenter kan importeres på to måder:

<code>{`import { Button } from '@norlys/norlys-components';`}</code>

eller

<code>{`import Button from '@norlys/norlys-components/button';`}</code>
